# StageLayout Designer

学生団体やイベント現場向けの、ステージ仕込み図作成アプリです。

音響機材、照明、舞台パーツ、人物、バミリ、ケーブル線、テキスト、PDF/画像の背景図面をキャンバス上に配置し、提出用プレビュー、PNG保存、PDF保存、印刷まで行えます。

Java / Swing で制作しています。

## 制作目的・制作意図

大学サークルや学生団体では、毎年メンバーが入れ替わるため、機材の扱いや図面作成に慣れていない人へ作業を引き継ぐ必要があります。

このアプリでは、機能を増やしすぎるよりも、機材を選んで配置する、必要機材を確認する、保存して次の担当者に渡す、といった基本操作を分かりやすくすることを重視しています。

このアプリでは、以下の内容が伝わることを目指しています。

- なぜその機材を置くのか
- どこに機材や出演者を配置するのか
- 当日必要な機材が何か
- 後輩や友人が見ても分かりやすい図面になっているか

高機能な作図ソフトを置き換えるものではなく、Dynamic Draw などより機能を絞り、初心者でも迷いにくい仕込み図作成ツールを目指しています。

## 制作背景

大学時代、放送部でイベント運営に関わる中で、ステージ上の機材配置、出演者の立ち位置、スピーカーや照明機材の場所などを分かりやすく共有することの大切さを感じました。

また、前職ではイベント現場に関わり、照明機材の設営や本番対応を経験しました。機材配置や事前準備の情報共有が、現場の動きやすさにつながることを実感しました。

そうした経験から、専門的な図面作成ツールに慣れていない人でも、視覚的に配置を確認できるアプリがあれば便利だと考え、制作を始めました。

## 実装した機能

### 機材配置

- 左側の機材パレットから機材を選択
- `放送部`、`すべて`、`マイリスト` の大きな切替ボタン
- `すべて` から右クリックで `放送部` または `マイリスト` に追加
- `放送部` と `マイリスト` は最初は空
- 追加した機材だけを、元のジャンル別に表示
- 音響、照明、舞台、人物などのカテゴリ表示
- 音響はスピーカー、マイク・スタンド、その他音響に整理
- 機材画像と表示名の表示
- ダブルクリックで機材を配置
- ドラッグ移動
- 回転ハンドルによる回転
- 拡大、縮小、削除
- Shiftクリックによる複数選択
- 複数選択した機材のまとめ移動、まとめ削除
- 名前変更、コピー、前面/背面移動
- 個別ラベル表示ON/OFF
- グリッド表示ON/OFF
- グリッド吸着ON/OFF
- 整列ガイドによる中心線・端への吸着補助

### 線・ケーブル線・導線

- 通常線、ケーブル線、導線、バミリ線を切り替え
- 線の開始はダブルクリック、開始後はクリックで連続描画
- 導線は終点に矢印を表示
- 線の色と太さを変更
- つながった線の種類をまとめて変更
- 線本体をドラッグして、つながった同じ種類の線をまとめて移動
- 始点・終点ハンドルをドラッグして長さや向きを調整
- 既存線の端点から続きの線を描画
- 線の右クリック削除、Delete削除
- バミリ線は赤いテープ位置として扱い、メートル表示なし

### バミリ

- 横、縦、X、十字、L、T、自由線に対応
- サイズ変更
- 回転
- 個別ラベル表示ON/OFF
- プレビューでは提出用に見やすい表示

### テキストボックス

- 図面上に任意の文字を配置
- 受付、音響卓、出演者待機、立入禁止などの補足説明に利用
- ダブルクリックで本文編集
- ドラッグ移動
- 右下ハンドルでサイズ変更
- 未選択時と提出用プレビューでは、基本的に文字だけ表示

### 背景図面・PDF読み込み

- PNG / JPG / JPEG / PDF形式のフロアマップを背景として読み込み
- PDFは先頭ページを背景図面として使用
- 読み込み時に実寸幅mを入力して、グリッドに合わせたサイズへ調整
- 背景図面の表示/非表示、固定/解除、削除
- 透明度調整
- シート幅合わせ、シート高さ合わせ、実寸幅合わせ、中央配置
- 読み込んだ背景図面を会場テンプレートとして登録

### 会場テンプレート

- `舞台 > 会場パーツ` から会場図を作成
- 長方形、半円前面、楕円形、陸上トラック風などの図形テンプレート作成
- 214教室、大学野外ステージなどの既存テンプレート
- 会場パーツと背景図面を会場テンプレートとして保存
- 保存した会場テンプレートの読み込み
- 会場固定による会場パーツの誤操作防止

### プレビュー・提出用出力

- 提出用プレビュー表示
- プレビュー範囲の切り替え
- プレビュー内の線の長さ表示ON/OFF
- 必要機材一覧の自動集計
- 必要機材一覧をできるだけ省略せず、列数や文字サイズで調整
- メモ欄表示
- PNG画像として保存
- PDFとして保存
- 印刷
- イベント名と日付をもとにした保存名

### 保存・読み込み

- 独自形式 `.stage` で保存
- イベント名、日付、会場名、担当者、メモを保存
- 機材の位置、サイズ、回転、ラベル表示を保存
- 会場パーツ、線、背景図面、テキストボックスを保存
- 今のファイルを上書き保存
- 会場だけ保存
- 会場テンプレート読み込み

### ヘルプ

- `ヘルプ > 使い方` から基本操作を確認
- 配布後もアプリ内だけで最低限の使い方を確認可能

## 起動方法

### 必要な環境

開発やコンパイルを行う場合は Java 21 が必要です。

配布版を使う場合は、Java runtime を同梱する想定のため、利用者側でJavaを入れる必要はありません。

### 配布版を使う場合

GitHub Releases から配布Zipをダウンロードします。

Zipを展開し、以下を実行します。

```text
StageLayoutDesigner.exe
```

### 開発環境でコンパイルする場合

このプロジェクトは Maven / Gradle を使わず、Java 21 の `javac` で直接コンパイルしています。

PowerShell例:

```powershell
javac -encoding UTF-8 -cp ".\lib\pdfbox-app-3.0.8.jar" -d ".\bin" (Get-ChildItem -Path ".\src" -Recurse -Filter "*.java" | ForEach-Object { $_.FullName })
```

画像などの resources を変更した場合は、`src\resources` を `bin\resources` にコピーします。

```powershell
Copy-Item -Path ".\src\resources" -Destination ".\bin" -Recurse -Force
```

起動例:

```powershell
java -cp ".\bin;.\lib\pdfbox-app-3.0.8.jar" app.Main
```

## PDFBoxについて

PDF形式のフロアマップを読み込むために Apache PDFBox を使用しています。

- Library: Apache PDFBox
- Version: 3.0.8
- File: `lib/pdfbox-app-3.0.8.jar`
- License: Apache License 2.0

ライセンス文およびNOTICEは、`licenses` フォルダに収録しています。

EclipseでPDFBoxが認識されない場合は、`lib/pdfbox-app-3.0.8.jar` を右クリックして `Build Path > Add to Build Path` を選択してください。

## 注意事項

このプロジェクトは現在、学習およびポートフォリオ用途で開発しています。

一部の機材シンボル画像に Dynamic Draw 関連のアプリ画像・素材を利用、または参考にしたものが含まれています。画像素材の権利は各権利者に帰属します。

再配布、商用利用、公開範囲の拡大を行う場合は、素材の利用条件を確認し、必要に応じて自作素材や利用許可のある素材へ差し替える必要があります。

また、本アプリでは利用者が所有または利用許可を得た画像やPDFを背景図面として読み込めます。ホールや施設の公式サイトで公開されているフロアマップを利用する場合は、各施設の利用条件を確認してください。

本リポジトリでは、第三者が権利を持つフロアマップ画像やPDFを同梱しません。
