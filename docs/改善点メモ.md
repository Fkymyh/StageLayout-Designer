# 改善点メモ

StageLayout Designer の現状を見て、今後直すとよさそうな点を優先度別に整理します。
今回は大規模なリファクタリングは行わず、読みやすくするためのコメント追加に留めています。

## 優先度高

- `CanvasPanel` に描画、選択、移動、リサイズ、線描画、バミリ、テキストボックス、背景図面、undo/redo が集中している。
  すぐ分割すると危険ですが、今後不具合が増えやすい場所なので、描画処理、入力処理、履歴処理、背景図面処理を少しずつ切り出す候補です。
- モード切替は改善されてきたものの、文字、線、バミリ、会場パーツ、機材配置の状態が `CanvasPanel` と `MainFrame` にまたがっている。
  使っている人から見たモード表示と内部状態がずれると混乱するため、今後も変更時は重点確認が必要です。
- `.stage` 保存形式に後から列が増えているため、読み込み時の互換処理が重要になっている。
  新しい保存項目を足す時は、古いファイルを読めるか必ず確認する必要があります。
- プレビューは提出用として重要なので、作業画面と同じ描画を流用しすぎると見づらくなる。
  テキストボックス、バミリ、必要機材一覧などは、プレビュー専用の見せ方を守る必要があります。
- 後輩テストでは、PDF背景、バミリ、文字、プレビューの方向性は良さそうでした。
  今後は新機能を増やすより、初見で迷う場所、掴みにくい場所、小さすぎる機材を優先して直すと効果が大きいです。
- スタンドマイクセットは、ブームスタンド単体の画像を流用すると誤解されます。
  追加するなら、マイクとスタンドをまとめて配置する「複合機材」として作るか、専用画像を用意してから登録する方が安全です。

## 優先度中

- 後輩テスト後の改善候補として、複数選択と整列機能があります。
  左揃え、中央揃え、右揃え、上下揃え、等間隔配置があると便利ですが、現在は複数選択の土台がないため、操作体系を決めてから追加する必要があります。
- まずは複数選択より先に、単体移動時の整列ガイドを追加しました。
  機材を動かす時に、ほかの機材の中心X・中心Y・左端・右端・上端・下端へ吸着し、薄い青いガイド線を表示します。
  次の段階では、等間隔配置や複数選択時の一括整列を検討できます。
- 右クリックメニューは、機材、文字ボックス、会場パーツ、背景図面へ広げました。
  回転やサイズ変更は将来PowerPoint風のハンドル操作へ寄せる方針なので、右クリックには入れすぎない方が分かりやすいです。
- 機材と会場パーツの選択枠は青系に整理しました。
  機材には選択枠の上側へ丸い回転ハンドルを追加しました。Shiftを押しながら回すと15度刻みになります。
- 文字ボックスの選択枠とリサイズハンドルも青系にそろえました。
  編集中の対象がどれか分かる一方で、提出用プレビューの見た目には影響しません。
- マイリストはアプリ全体のよく使う機材一覧として使えます。
  将来的には、各ジャンル内で「すべて / マイリスト」を切り替えられるようにすると、照明など機材数が多いジャンルでもさらに探しやすくなります。
- 回転した機材は、見た目と選択判定が完全には一致しない場合があります。
  回転後も掴みやすくするには、回転後の形に合わせた当たり判定へ直す必要があります。
- 線機能は、直線、バミリ線、端点編集が中心です。
  曲線、自由線、線種、重なった線の選択、途中だけ消す操作は、今後の線編集機能として整理する候補です。
- 必要機材一覧は文字中心です。
  将来的には、画像、機材名、数量を並べる表示モードを追加すると、提出用としてさらに分かりやすくできます。
- 図形テンプレートは、文字だけの説明よりプレビューがある方が分かりやすいです。
  現在は小さな図形プレビューを出していますが、将来的には実際の配置後サイズ感やステージ名の表示も確認できるとさらに親切です。
- テンプレート、背景図面、会場パーツの関係が少し複雑になっている。
  214教室などの標準テンプレート、PDF/画像背景テンプレート、ユーザー保存テンプレートを、利用者には同じ「会場の土台」として見せる整理が必要です。
- 必要機材一覧は全件表示に寄せていますが、件数が多い時は紙面内で読みづらくなる可能性がある。
  将来的には2ページ目や別紙出力も検討できます。
- `RoomObject` が四角、円、画像、楕円、トラック風などをtypeで持っている。
  形状が増え続ける場合は、描画だけを別クラスに分けると読みやすくなります。
- PDF/画像背景は元ファイルパスに依存している。
  別PCで開く時や配布時に背景が見つからない可能性があるため、テンプレート保存時のコピー方針を今後さらに整理すると安全です。
- ツールバーとメニューは整理されてきたが、機能が増えると再び入口が増えすぎる可能性がある。
  初心者向けには「よく使う操作」と「管理操作」を分ける方針を維持するとよいです。

## 優先度低

- 213教室テンプレートは、寸法や常設物の位置が分かる資料がそろってから追加する方が安全です。
  推測で作ると、後輩が実際の会場図として信じてしまう可能性があります。
- 機材画像は、名前と見た目が一致していることを優先します。
  代用画像を使う場合は、後輩が見ても誤解しない名前にするか、追加を見送る方がよいです。
- UIの色、余白、ボタン名をさらに統一すると、アプリ全体の見た目が落ち着きます。
- コード製テンプレートの 214教室、大学野外ステージは便利ですが、将来的にはユーザー保存テンプレートへ寄せると引き継ぎやすくなります。
- 現在は手動コンパイル中心なので、将来的に簡単なビルドスクリプトを用意すると配布作業が楽になります。
- PDFBoxなど外部ライブラリのライセンス確認手順を、配布前チェックリストにしておくと安心です。
