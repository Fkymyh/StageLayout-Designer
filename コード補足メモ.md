# StageLayout Designer コード補足メモ

このメモは、後輩や友人が「どこを見れば何が分かるか」を探しやすくするための補足です。
細かい実装を全部覚えるより、まず画面、データ、保存、配布のつながりを把握することを目的にしています。

## 全体の流れ

アプリの開始地点は `src/app/Main.java` です。
ここから `view.MainFrame` を開き、メイン画面全体を組み立てています。

基本的な流れは次の通りです。

1. `Main` が起動する。
2. `MainFrame` が画面、メニュー、ツールバーを作る。
3. 左の `EquipmentPanel` で機材を選ぶ。
4. 中央の `CanvasPanel` に機材や会場パーツを置く。
5. 右の `PropertyPanel` で選択中の機材情報を編集する。
6. `LayoutFileManager` が `.stage` ファイルとして保存・読み込みする。
7. `PreviewDialog` と `SheetPreviewPanel` が提出用プレビューを表示する。

## よく触るファイル

### `src/view/MainFrame.java`

メイン画面の中心です。
メニュー、ツールバー、左パレット、キャンバス、右プロパティ欄をまとめています。

ここを触ることが多い例:

- メニュー項目を増やす
- ツールバーのボタンを増やす、減らす
- 保存、開く、新規作成、プレビューの動きを変える
- 会場テンプレートの読み込み導線を変える
- 新規作成時の作業シートサイズ候補を変える

### `src/view/CanvasPanel.java`

作業シートそのものです。
機材の表示、ドラッグ移動、選択、回転、サイズ変更、グリッド、余白、会場固定、戻る・やり直しなどを扱っています。

ここを触ることが多い例:

- キャンバス上の見た目を変える
- グリッドや余白の見え方を変える
- 機材を置く、動かす、消す処理を変える
- 会場固定時に動かせない対象を変える
- 戻る、やり直しの対象を調整する

注意点:

- 作業シートの外側に見える余白は、見やすさのための余白です。
- グリッドやメートル表示は、余白ではなく作業シート本体に合わせています。
- `roomTemplate` はコードで用意された既存テンプレート、`customRoomObjects` はパレットから作った会場パーツです。

### `src/view/EquipmentPanel.java`

左側の機材パレットです。
ジャンルごとの表示、お気に入り、機材ボタンの表示を扱っています。

ここを触ることが多い例:

- パレットの見やすさを変える
- お気に入りの表示を変える
- 機材ボタンの並びや大きさを変える

お気に入りはプロジェクトごとではなく、このPC全体で共通です。

### `src/model/EquipmentFactory.java`

アプリに登録されている機材一覧です。
機材名、カテゴリ、初期サイズ、画像パスをここで登録しています。

ここを触ることが多い例:

- 新しい機材を追加する
- 音響、照明、舞台などのカテゴリを変える
- 初期サイズや画像を変える

注意点:

- 保存ファイルでは機材名を使って復元しています。
- すでに保存済みの `.stage` ファイルがある場合、機材名を急に変えると読み込み時に別物扱いになりやすいです。

### `src/model/RoomTemplateFactory.java`

コードで最初から用意している会場テンプレートです。
現在は 214教室や大学野外ステージなどをここで作っています。

ただし今後の方針としては、後輩がコードを書かなくて済むように、パレットから会場パーツを置いて `ファイル > 会場だけ保存` する方法を基本にします。

### `src/io/LayoutFileManager.java`

`.stage` ファイルの保存と読み込みを担当します。
プロジェクト情報、機材、会場パーツ、線をテキストとして保存しています。

ここを触ることが多い例:

- 保存したい情報を増やす
- 古い保存ファイルも読めるようにする
- 会場テンプレート保存の中身を調整する

注意点:

- 保存形式を変える時は、昔作ったファイルが読めなくならないか確認します。
- カンマや改行を含むメモ欄が壊れないように、簡単なエスケープ処理をしています。

### `src/view/PreviewDialog.java` と `src/view/SheetPreviewPanel.java`

提出用プレビューを表示する画面です。
画面上の作業シートとは別に、印刷や画像保存向けの見た目で描画しています。

ここを触ることが多い例:

- プレビューの文字サイズや余白を変える
- 必要機材一覧の表示を変える
- PNG保存時の見た目を整える

## データの考え方

### 背景図面

背景図面は `BackgroundMap` で管理します。
会場PDFや画像を下絵として表示するための機能で、CanvasPanelでは用紙背景の上、会場パーツや機材の下に描画します。

現在はPNG/JPG/JPEGを `ImageIO` で読み込みます。
PDFはJava標準だけでは画像化できないため、PDFBoxを追加して1ページ目を画像に変換する予定です。

### テキストボックス

テキストボックスは `TextBoxItem` で管理します。
機材に紐づかない説明や注意書きを図面上に置くために使います。
CanvasPanelでは機材や線より上に描画します。

### 機材

機材の元データは `Equipment` です。
キャンバスに置かれた機材は `LayoutItem` になります。

`LayoutItem` には、位置、サイズ、回転、数量、メモなどがあります。
同じ「マイク」でも、置かれた場所ごとに別々の `LayoutItem` として扱います。

### 会場パーツ

会場パーツは `RoomObject` です。
四角形、円、線、画像などを使って会場の形や常設物を表します。

パレットから作った会場パーツは `customRoomObjects` に入ります。
`会場固定` がONの時は、会場パーツを動かしにくくして、当日の機材配置だけに集中できるようにしています。

### 線

ケーブルや矢印などの線は `DrawLine` です。
始点、終点、色、太さを持っています。

## 保存ファイル

通常の保存ファイルは `.stage` です。
中身はテキストで、主に次のブロックに分かれています。

- `#PROJECT`: イベント名、日付、会場、担当者、メモ
- `#ITEMS`: 配置した機材
- `#ROOM_OBJECTS`: パレットから作った会場パーツ
- `#LINES`: 線

会場だけ保存する時は、機材と線を空にして、会場パーツだけを保存しています。
読み込む時は、現在の機材配置を残したまま会場パーツだけを入れ替えます。

## 画像と素材

画像は主に `src/resources/images` に入っています。
機材として使う画像は `EquipmentFactory` の画像パスから読み込まれます。

画像を増やした時は、コンパイル後に `src/resources` を `bin/resources` にコピーする必要があります。

## コンパイル

このプロジェクトは Maven や Gradle ではなく、VS Code 拡張に入っている Java の `javac` を直接使ってコンパイルしてきました。

基本は次の考え方です。

- `src` 以下の `.java` を全部コンパイルする
- 出力先は `bin`
- 文字コードは `UTF-8`
- 画像などを変えたら `src/resources` を `bin/resources` にコピーする

## 配布方針

Java が入っていないPCへ渡すなら、最終的には `jpackage` で Windows 用アプリにするのが向いています。
`jar` だけだと、相手のPCに Java が入っていない場合は起動できません。

そのため、友人や後輩にテストしてもらう段階では次の順番がおすすめです。

1. まず `.jar` を作って、自分のPCで起動確認する。
2. 次に `jpackage` で Java 付きの配布用フォルダ、またはインストーラーを作る。
3. Java が入っていない別PCで起動確認する。

## 今後触るなら

今の方針では、機能を増やすより「迷わず使えること」を優先します。

優先度が高いもの:

- 会場テンプレートをパレットから作りやすくする
- プレビューを提出用として見やすくする
- 配布しやすい形にする
- 後輩が操作で迷った場所を直す

優先度を下げてよいもの:

- ログイン機能
- ジャンル別の複雑な画面切り替え
- プロ向け作図ソフトのような細かすぎる機能
